# 阅读目录助手

## 目标

将 `raw/01-articles/hello-agents/docs/chapterN/` 下的中文主 Markdown 编译为一份可直接查询的章节阅读目录。目录回答“每一小节讲了什么”，不输出“抓住的点”“学习建议”或额外延伸。

## 生成流程

1. 读取 `wiki/index.md`，确认已有目录与关联概念。
2. 定位章节目录中的中文主文件：优先匹配 `第N章 *.md`，排除英文副本和 `raw/09-archive/`。
3. 完整阅读源文件，包括正文和代码示例；提取 `##` 与 `###` 标题，不将代码内的注释标题当作章节小节。
4. 为每个二级、三级小节分别编写内容，并按二级小节分组：
   - 每个二级小节使用 `### N.M 标题` 作为独立区块，标题下直接写该主题的整体说明；三级小节只写自身内容，不能合并进父级。
   - 在二级小节概述之后，创建仅包含其直属三级小节的独立表格；三级小节必须紧跟所属二级标题。没有三级小节时，不创建空表格。
   - “讲了什么”使用简体中文，目标 180～250 字，绝不超过 350 字。
   - 说明概念、过程、案例、边界与章节结论；不增加源文件没有的结论。
   - 保留必要的英文术语、类名或协议名；不要逐行复述代码。
5. 创建或更新 `wiki/syntheses/hello-agents-chapterN-reading-catalog.md`，使用以下结构：

```markdown
---
title: "hello-agents-chapterN-reading-catalog"
type: synthesis
tags: [阅读目录, 学习笔记, 智能体, HelloAgents]
sources: [raw/01-articles/hello-agents/docs/chapterN/第N章 文件名.md]
last_updated: YYYY-MM-DD
---

## 问题

第 N 章的每个小节分别讲了什么？

## 阅读目录

### N.1 标题

约 180～250 字的二级小节整体说明。

| 小节 | 讲了什么 |
| --- | --- |
| N.1.1 标题 | 约 180～250 字的三级小节内容说明。 |

## 使用约定

- 询问“第 N 章讲了什么”时，直接使用本目录中全部二级区块及其子小节表格回答。
- 询问“N.M 讲了什么”时，只返回该二级标题下的整体说明；询问“N.M.P 讲了什么”时，只返回对应表格行的“讲了什么”内容，不再改写、拆点或补充“抓住的点”。

## 关联连接

- [[摘要-hello-agents-chapterN]] — 章节来源摘要。
```

6. 在 `wiki/index.md` 的 `Syntheses` 分类注册目录页；在 `wiki/log.md` 追加 `sync` 记录。若章节已有来源摘要、课程地图或概念页，为目录页补充有效双链。

## 查询流程

当用户问已生成章节的概览或具体小节时：

1. 先读取 `wiki/index.md` 和对应 `hello-agents-chapterN-reading-catalog.md`。
2. 问“第 N 章讲了什么”时，直接返回全部二级区块与子小节表格；问二级小节时，仅返回其整体说明；问三级小节时，仅返回该表格行的“讲了什么”单元格内容。
3. 不重新总结为更短版本，不增加“要点”“建议”列，也不修改目录文件，除非用户明确要求更新。

## 约束与校验

- 不修改 `raw/` 中任何原始文件。
- 只读写当前知识库根目录内的 `wiki/` 与 `.agents/skills/`。
- 每次生成后检查：Frontmatter、`## 关联连接`、来源路径、双链、`wiki/index.md` 注册以及 `wiki/log.md` 追加记录。
- 若找不到中文主章节、目录标题与正文结构矛盾，或旧目录与新源内容发生实质冲突，暂停并向用户说明。
