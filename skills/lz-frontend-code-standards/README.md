# lz-frontend-code-standards

## 用途

这个 skill 用于在前端实现阶段统一约束 Vue、JavaScript、TypeScript 的新增和修改代码规范。

## 使用场景

- 开始编写或修改 `.vue` 文件
- 开始编写或修改 `.js`、`.ts` 文件
- 开始做组件开发、页面开发、前端重构
- 开始实现前端交互逻辑
- 开始编写或修改 composables / hooks

以下情况不触发：

- 纯分析
- 纯讨论
- 纯方案设计

## 约束边界

- 不主动治理历史旧代码
- 只对本次新增和修改的代码强制执行规范
- 先识别项目当前 Vue 版本、API 风格和主流写法，再在改动范围内收敛规范

## 覆盖内容

- JavaScript / TypeScript 通用命名与代码风格
- 接口响应严格按已确认契约取值，禁止无依据默认值、跨字段兜底和猜测性兼容
- 仅在需求明确时允许多版本兼容，并要求注明依据、字段差异和移除条件
- Vue 基础规范
- `on` / `handle` 事件命名规范
- `xxxRef` 命名规范
- 表单 `options` 结构规范
- `<script setup>` 固定顺序
- hooks / composables 拆分规范
- Review 友好性规则
- 写代码前检查项
- 提交前自检项

## 触发确认

当这个 skill 在前端实现阶段真正触发时，应先输出一行固定标记：

`[已触发 lz-frontend-code-standards]`

如果任务只是分析、讨论或方案设计，则不应输出这行标记。

## 示例

### 示例 1

输入：新增一个 Vue 组件并实现表单交互逻辑

预期行为：触发 skill，先识别项目风格，再按规范组织 `props`、状态、方法、事件命名和模板写法

### 示例 2

输入：修改一个 TypeScript 页面文件，补充列表筛选和接口调用逻辑

预期行为：触发 skill，对新增和修改的命名、临时变量、响应数据命名、早返回和结构顺序进行约束

### 示例 3

输入：接口字段已经确认为 `protocolName`、`protocolUrl`，修改协议列表映射

预期行为：直接使用已确认字段，不得自行添加 `name || protocolName`、`url || protocolUrl` 或空值兜底
