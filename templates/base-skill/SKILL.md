# 目的

说明这个 skill 帮模型完成什么任务。

## 何时使用

- 写清楚哪些用户请求应该触发这个 skill
- 写出一些相邻表述或上下文，帮助模型更稳定地触发
- 如果有明显不适用场景，也可以顺手写出来，避免误用

## 工作流

1. 理解用户请求，并收集完成任务所需的最小上下文。
2. 按这个 skill 的核心工作流执行任务。
3. 以稳定、可预期的结构给出结果。

## 输出要求

- 说明结果必须包含什么
- 说明输出格式或结构要求

## 备注

- 只有真的需要时才写工具依赖
- 说明要尽量通用，避免只对单个例子有效
