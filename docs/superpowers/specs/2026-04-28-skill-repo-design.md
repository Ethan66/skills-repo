# Skill 仓库设计

**目标：** 建立一个统一管理自定义 skill 的仓库，先保持轻量结构，同时为后续测试、打包、发布流程预留扩展空间。

## 选定结构

仓库采用低耦合的 monorepo 结构：

- `skills/` 存放真实 skill
- `templates/base-skill/` 存放新建 skill 的基础模板
- `docs/` 存放仓库规范和设计文档
- `dist/` 预留给后续打包产物

## 这样设计的原因

- skill 之间保持独立，便于分别维护
- 仓库足够简单，适合个人长期使用
- 后续增加校验、打包、发布时不需要推翻目录结构
- 同一套布局可以兼容 Codex、Claude Code 和通用 skill 场景

## 单个 Skill 的基础约定

每个 skill 初始至少包含三个文件：

- `SKILL.md`
- `README.md`
- `skill.json`

这样既保持轻量，也能为后续工具化保留必要元数据。

## 暂缓处理的内容

首次初始化阶段不包含以下内容：

- 自动化测试
- 打包脚本
- 发布脚本

这些能力等仓库中出现多个真实 skill 后再逐步补齐。
