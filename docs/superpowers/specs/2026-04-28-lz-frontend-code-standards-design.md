# lz-frontend-code-standards 设计

**目标：** 创建一个在前端实现阶段自动触发的 skill，用于约束 Vue / JavaScript / TypeScript 新增和修改代码的写法与结构。

## 触发条件

只有在明确进入前端写代码或改代码阶段时触发，纯分析、纯讨论、纯方案设计阶段不触发。

当任务开始实施，且包含以下任一信号时自动触发：

- `.vue` 文件改动
- `.js` 或 `.ts` 文件改动
- 组件开发
- 页面开发
- 前端重构
- 前端交互逻辑实现
- composables / hooks 编写或修改

## 执行边界

- 不主动治理旧代码
- 只对本次新增和修改的代码强制执行规范
- 先识别项目当前 Vue 版本、API 风格和主流写法
- 在不大面积重构旧代码的前提下，让新增和修改代码按规范收敛

## 规范范围

- JavaScript / TypeScript 通用命名与风格规范
- Vue 基础规范
- `on` / `handle` 事件命名分工
- `xxxRef` 命名规则
- 表单 `options` 结构规则
- `<script setup>` 固定顺序
- hooks / composables 拆分规则
- Review 友好性规则
- 写代码前检查项
- 提交前自检项

## 设计原则

- 规范要能直接用于实现阶段，而不是停留在文档说明
- 规则尽量具体，减少模糊空间
- 以跟随项目现状为前提，但对新增和修改代码保持强约束
